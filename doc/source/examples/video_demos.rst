Video demos
===========

Use short videos to show common MCP workflows end to end.

Recommended demo set
--------------------

- Connection flow: ``check_aedt_status`` -> ``launch_aedt`` ->
  ``create_design``.
- Existing-session flow: ``connect_to_aedt`` with a running AEDT instance.
- Solve and export flow: ``analyze_design`` -> ``export_results``.
- Debug flow: ``get_pyaedt_logs`` and small-step ``run_python_code`` retry.

Suggested structure per video
-----------------------------

1. Goal (one sentence).
2. Input state (AEDT already running or not, project available or not).
3. Tool call sequence.
4. Expected output artifacts (for example, Touchstone file or screenshot).

Recording tips
--------------

- Keep each video under 2 to 3 minutes.
- Show exact tool names and key arguments.
- Include one failure-and-recovery moment when possible.
- End with generated files or plots visible in the workspace.

Where to reference demo videos
------------------------------

Add links to published videos from this page when available.
